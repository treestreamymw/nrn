5 /* In order to support oc objects containing sections, instead of vector
6  of ordered sections, we now have a list (in the nmodl sense)
7  of unordered sections. The lesser efficiency is ok because the
8  number crunching is vectorized. ie only the user interface deals
9  with sections and that needs to be convenient
10 */
11 /* Data structure for solving branching 1-D tree diffusion type equations.
12  Vector of ordered sections each of which points to a vector of nodes.
13  Each section must have at least one node. There may be 0 sections.
14  The order of last node to first node is used in triangularization.
15  First node to last is used in back substitution.
16  The first node of a section is connected to some node of a section
17  with lesser index.
18 */
19 /* An equation is associated with each node. d and rhs are the diagonal and
20  right hand side respectively. a is the effect of this node on the parent
21  node's equation. b is the effect of the parent node on this node's
22  equation.
23  d is assumed to be non-zero.
24  d and rhs is calculated from the property list.
25 */

237 /* a point process is computed just like regular mechanisms. Ie it appears
238 in the property list whose type specifies which allocation, current, and
239 state functions to call. This means some nodes have more properties than
240 other nodes even in the same section. The Point_process structure allows
241 the interface to hoc variable names.
242 Each variable symbol u.rng->type refers to the point process mechanism.
243 The variable is treated as a vector
244 variable whose first index specifies "which one" of that mechanisms insertion
245 points we are talking about. Finally the variable u.rng->index tells us
246 where in the p-array to look. The number of point_process vectors is the
247 number of different point process types. This is different from the
248 mechanism type which enumerates all mechanisms including the point_processes.
249 It is the responsibility of create_point_process to set up the vectors and
250 fill in the symbol information. However only after the process is given
251 a location can the variables be set or accessed. This is because the
252 allocation function may have to connect to some ionic parameters and the
253 process exists primarily as a property of a node.
254 */
