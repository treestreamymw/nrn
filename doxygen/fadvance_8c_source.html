16 /*
17  after an fadvance from t-dt to t, v is defined at t
18  states that depend on v are defined at t+dt/2
19  states, such as concentration that depend on current are defined at t
20  and currents are defined at t-dt/2.
21  */
22 /*
23  fcurrent is used to set up all assigned variables without changing any
24  states. It assumes all states have their values at time t which is
25  only first order correct. It determines the nonvint assignments first
26  so that ena, etc. are correct for the current determinations. We
27  demand that nonvint assignments can be done without changing states when
28  dt is temporarily set to 0.
29 
30  It turned out to be a bad idea to do this. Many methods (KINETIC, PROCEDURE)
31  for solving are not even called when dt =0. Also it plays havoc with tables
32  that are recomputed several times when dt is changed then changed back.
33  Therefore we are no longer trying to initialize assigned variables within
34  SOLVE'd blocks. Instead we are making initialization more convenient,
35  at least within the normal situations, by introducing a finitialize()
36  with an optional argument vinit. finitialize will call the INITIALIZE block
37  for all mechanisms in all segments. (Default initialization sets all states
38  to state0.) The user can set things up specially in a models INITIAL
39  block. INITIAL blocks can make use of v. With care they can make use of
40  ionic concentrations just like breakpoint and solve blocks. When the argument
41  is present, v for all segments are set to that value.
42  */
